# Responsible AI Use Policy

## Purpose

This policy provides basic responsible-use guidance for employees using Microsoft 365 Copilot and approved AI tools.

## Approved Use Examples

Employees may use approved AI tools to:

- Draft internal communications
- Summarize meeting notes
- Create outlines
- Organize project tasks
- Improve clarity of writing
- Generate training materials
- Review public or approved internal content

## Restricted Use Examples

Employees should not use AI tools to:

- Share passwords
- Share protected personal information
- Upload confidential client data without approval
- Make final legal, financial, HR, or compliance decisions
- Bypass company security policies
- Generate misleading or unverified information

## Data Protection Guidance

Employees should remember:

- Copilot only surfaces content the user already has permission to access
- Poor permission management can create risk
- Sensitive information should be handled carefully
- AI output should always be reviewed
- Access issues should be escalated to IT

## Escalation

Employees should contact IT support when:

- Copilot shows unexpected data
- Access permissions appear incorrect
- AI-generated content seems inaccurate
- A sensitive information issue is suspected
- An employee is unsure whether a use case is approved
